Choix techniques

Node.Js


Dans le cadre de ce projet en NoSQL database, nous avons choisi la plateforme logicielle Node.Js pour réaliser notre application en réseau. Elle utilise la machine virtuelle V8 et permet de monter en local un serveur.

Javascript, HTML & Css

Le Javascript nous permet d'utiliser la programmation orientée objet pour créer des interactions entre l'utilisateur d'une page de l'application et cette dernière. Pour coder les pages de l'application, nous avons utiliser HTML et CSS complété d'un framework " BootStrap" pour accéder rrapidemment à un design agréable.

Les requêtes

Notre application permet d'intéragir sur une base de données contenant des citations, les quotes. L'utilisateur peut en ajouter, les consulter, ou bien les supprimer.

Structure du code

L'application se structre grâce à une page d'acceuil avec toutes les fonctionnalités dessus afin de proposer une "One Page" application. Nous utilisons une seconde page seulement pour expliquer nos choix ainsi que la démarche utilisée pour structurer notre code.
Nous avons trois pages qui nous permettent de gérer l'ajout, l'affichage ainsi que la suppression des quotes. Nous avons structuré notre code afin de répondre aux attentes de l'énoncé dans le même orde.
Notre base de données Cities est défini grâce au JSON. Celui-ci nous permet de transferer la base de données par réseau ou entre deux machines sans perte d'information et sans problème de codage/ décodage. Ainsi utiliser NoSQL nous permet de traiter à grande échelle l'ensemble des données de la base Cities.
